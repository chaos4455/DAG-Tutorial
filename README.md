## ⚙️ Conceitos Básicos

Aqui, você encontrará uma introdução completa aos principais termos e conceitos necessários para entender o que são Grafos Acíclicos e como eles se relacionam com a evolução dos **LLMs**:

- **Grafos Acíclicos Dirigidos (DAGs)**: Estruturas de dados que modelam dependências sem loops.
- **LLMs (Large Language Models)**: Modelos de linguagem de grande escala, como GPT, BERT e seus derivados.
- **RAG (Retrieval-Augmented Generation)**: Técnica avançada que combina recuperação de dados com geração de linguagem.
- **Context Windows**: Janelas de contexto que controlam a quantidade de dados que o modelo pode processar de uma vez.

## 🧩 Grafos Acíclicos no Contexto de LLMs

Os **Grafos Acíclicos Dirigidos (DAGs)** desempenham um papel fundamental ao fornecer uma maneira eficaz de representar as relações de dependência entre diferentes tokens e inputs durante o treinamento e a inferência de **LLMs**. Este repositório explora a **estrutura de grafos** como um framework que pode ser aproveitado para **otimização de janelas de contexto** e na **seleção de informações relevantes** em um cenário de **retrieval**.

## 🔍 Retrieval-Augmented Generation (RAG)

### O que é RAG?

RAG combina a **recuperação de informações** com a geração de linguagem, permitindo que modelos de IA como **GPT** e **BERT** melhorem suas respostas com dados recuperados de fontes externas, como documentos ou bancos de dados. 

**Por que usar RAG?**
- 📊 **Acesso a informações atualizadas** em tempo real.
- 🔄 **Geração de respostas contextualmente relevantes**.
- 🚀 **Aprimoramento da precisão** em tarefas como question answering e sumarização de textos.

---

## 🖼️ Janelas de Contexto (Context Windows)

A **janela de contexto** de um modelo determina quantos tokens ele pode processar simultaneamente. Entender e **manipular de forma eficiente** essa janela é crucial para a **otimização de performance** de LLMs em diferentes tarefas. Neste repositório, você aprenderá a:

- 📐 Ajustar dinamicamente o tamanho das janelas.
- 🧩 Utilizar DAGs para melhorar a **seleção de tokens**.
- 🔍 **Maximizar o contexto** sem ultrapassar os limites do modelo.
